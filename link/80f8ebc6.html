简单使用

​ 使用 Ray Tune 进行超参数搜索非常方便，并且具有良好的可扩展性和并行化能力。同时，Tune 还支持多种搜索算法和调参策略，可以帮助用户更快地找到最优的超参数组合。

config

​ 在 Ray Tune 中，config 字典可以包含各种类型的参数，包括：

  • float：浮点型参数。
  • int：整型参数。
  • bool：布尔型参数。
  • string：字符串型参数。
  • choice：离散型参数，可以是一个列表，从中随机选择一个元素作为参数值。
  • quniform：均匀分布型参数，需要指定参数的下限、上限和间隔。
  • qloguniform：对数均匀分布型参数，需要指定参数的下限、上限和间隔。
  • qnormal：正态分布型参数，需要指定参数的均值、标准差和间隔。
  • loguniform：对数均匀分布型参数，需要指定参数的下限和上限。

​ 除了上述类型之外，用户还可以自定义其他类型的参数。在定义参数类型时，需要使用相应的函数或类来指定参数的类型和范围。

​ 需要注意的是，在使用 Ray Tune 进行超参数搜索时，不同参数类型的定义方式可能有所不同，具体要根据不同的搜索算法和调参策略来决定。例如，在网格搜索中，只能使用离散型参数和枚举参数，而在贝叶斯优化中，可以使用更多的参数类型和分布。

​ 在实际应用中，需要根据具体问题和搜索目标来选择合适的参数类型和范围，以及相应的搜索算法和调参策略，以获得最优的超参数组合。

turn.run()

​ 在 Ray Tune 中，tune.run() 函数是执行超参数搜索的入口函数，它有许多可配置的参数，用于控制搜索过程和评估结果的统计方法。下面是 tune.run() 函数中常用的一些参数：

  • run_or_experiment：必选参数，指定要执行的训练任务或实验。可以是一个训练函数，也可以是一个 Ray Tune 可识别的实验配置（如一个 JSON 或 YAML 文件）。
  • config：必选参数，定义了超参数搜索的空间和范围。它应该是一个字典，包含要搜索的超参数及其可能的取值范围。
  • stop：必选参数，指定了停止搜索的条件，可以是一个训练轮数、时间限制或达到一定的性能阈值。
  • num_samples：可选参数，指定搜索过程中尝试的不同超参数组合的数量。默认值为1。
  • resources_per_trial：可选参数，指定每个试验使用的资源，如 CPU、GPU、内存等。
  • num_gpus：可选参数，指定每个试验使用的 GPU 数量。
  • num_cpus：可选参数，指定每个试验使用的 CPU 数量。
  • local_dir：可选参数，指定本地存储超参数搜索结果的目录。
  • checkpoint_freq：可选参数，指定在训练过程中保存检查点的频率。
  • keep_checkpoints_num：可选参数，指定保留的检查点数。
  • checkpoint_score_attr：可选参数，指定评估训练任务性能的指标名称，用于选择最优的检查点。
  • max_failures：可选参数，指定超参数搜索过程中允许的最大失败次数。

​ 除了上述参数之外，tune.run() 函数还有许多其他可选参数，例如中断处理、调试模式、并行度控制等。用户可以根据自己的需求和搜索目标选择合适的参数，并进行适当的调参和优化，以获得最佳的超参数组合。

除了上面提到的常用参数，Ray Tune 的 tune.run() 函数还提供了一些其他可选参数，下面我们逐一介绍：

  • scheduler: 调度器。默认使用ASHAScheduler, 也可以设置为其他调度器，如 AsyncHyperBandScheduler、MedianStoppingRule等。
  • name: 针对该试验的名称，用于标识该试验的实例。如果不指定，则 Ray Tune 将自动生成一个唯一名称。
  • trial_name_creator: 用于创建每个试验名称的回调函数。如果未设置，则会使用 Ray Tune 提供的默认方法来生成名称。
  • log_to_file: 是否将结果日志输出到文件中，默认为True。
  • loggers: 一个或多个logger，用于记录结果日志。支持TensorBoardLogger、JSONLogger、CSVLogger、MLFlowLogger等不同类型的logger。
  • resume: 是否从之前的 checkpoint 继续运行搜索。
  • queue_trials: 是否将试验排队，默认为 False。如果为True，则试验将按顺序运行，而不是并发运行。
  • reuse_actors: 是否复用已有的 actors 来启动新的试验。
  • trial_executor: 用于执行训练任务的执行器，默认为 RayTrialExecutor。也可以设置为其他的执行器。
  • raise_on_failed_trial: 是否在试验失败时引发异常。默认为 True。
  • search_alg: 搜索算法。默认为 HyperOptSearch，支持其他搜索算法如 RandomSearch、GridSearch等。
  • scheduler: 调度器。默认为 ASHAScheduler，支持其他调度器如 AsyncHyperBandScheduler、MedianStoppingRule等。
  • search_space: 搜索空间。用于指定搜索算法的参数空间。

​ 以上就是 Ray Tune 中 tune.run() 函数的所有可选参数，根据具体需求和搜索目标，用户可以选择不同的参数组合和配置方式，以获得最佳的超参数组合。

​ 在 Ray Tune 中，调度器（scheduler）是一种可以根据试验结果调整超参数搜索策略的工具。调度器可以根据不同的策略动态地调整试验的超参数、分配计算资源等，从而加速试验的收敛速度或提高试验结果的质量。Ray Tune 支持多种调度器，例如 HyperBand、ASHA 等。

​ HyperBand 是一种基于倍增策略的调度器，它会按照指数级别的规模扩展并行的试验，然后通过不断淘汰较差的试验和继续放大优秀试验的规模，来提高试验的效率。HyperBand 的主要优点是快速收敛、高效利用资源，但也有一定的缺点，比如可能会忽略一些潜在的好结果、试验结果具有一定的随机性等。

​ ASHA（Asynchronous Successive Halving Algorithm）是一种异步的连续二分调度器，它会按照指数级别的规模扩展并行的试验，并在每个规模下使用连续二分法（Successive Halving Algorithm）来剪枝较差的试验，从而提高试验的效率。ASHA 的主要优点是能够在较短的时间内找到接近最优的结果、具有较好的鲁棒性、支持动态分配资源等。

analysis

​ Ray Tune 的 Analysis 对象提供了丰富的 API，可以获取更多的统计信息，帮助用户深入了解超参数搜索的过程和结果。下面列举一些常用的 API：

  • dataframe(): 将结果转换为 pandas DataFrame 格式。
  • trials_dataframes(): 将每个试验结果转换为 pandas DataFrame 格式，并返回一个字典，字典的 key 为试验 ID，value 为 pandas DataFrame 格式的结果数据。
  • get_best_config(metric, mode='max'): 获取最佳超参数配置。metric 表示评估指标的名称，mode 表示评估模式，有两种可选值：max 和 min，分别表示评估指标的最大值和最小值。
  • get_best_trial(metric, mode='max'): 获取性能最佳的试验。与 get_best_config 类似，不过返回的是最优试验的结果数据。
  • get_trial(trial_id): 获取指定试验 ID 的结果数据。
  • get_best_logdir(metric, mode='max'): 获取性能最佳的试验的日志目录。
  • get_trainable_stats(trial_id, include_ray=False): 获取指定试验的统计信息，包括训练时长、训练进度、资源占用情况等。
  • get_all_stats(metric, mode='max'): 获取所有试验的统计信息，包括训练时长、训练进度、资源占用情况等。
  • last_result(trial): 获取最后一次试验结果的数据，包括评估指标、参数配置等。

​ 除了上面列举的 API，Analysis 对象还提供了其他一些方法，例如获取调度器状态、打印试验结果摘要等。通过这些 API，用户可以更加全面地了解超参数搜索的情况，并进行进一步的分析和优化。
